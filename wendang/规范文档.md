## 对齐规范：

对齐可分为水平对齐和垂直对齐两种基础逻辑，UI元素应当根据需要选择合适的对齐方式，在其所属容器内进行定位。

1. 水平对齐：分为左、右、居中和延展（Stretch）四种方式。
2. 垂直对齐：分为顶、底、居中和延展四种方式。
3. 数值对齐：为快速对比数值大小，所有数值取相同有效位数，并采用右对齐。

### 消息提示（Message / Toast）：

1. Message（全局提示）：位于页面顶部居中，2秒后自动消失，用于操作反馈。
2. Toast（轻提示）：位于底部或中央，1.5\~2秒后自动消失。
3. Notification（通知）：位于屏幕右上角，4\~5秒后自动消失，用于系统通知。

## 交互与反馈：

### 反馈机制：

| 场景   | 反馈方式          | 说明            |
| :--- | :------------ | :------------ |
| 操作成功 | Message       | 绿色提示，显示“操作成功” |
| 操作失败 | Message       | 红色提示，显示具体错误信息 |
| 加载中  | Spin          | 数据加载时展示       |
| 需确认  | Modal（Dialog） | 重要操作前二次确认     |
| 后台操作 | Loading按钮     | 提交类操作按钮置为加载状态 |

### 加载策略：

- 骨架屏（Skeleton）：页面初次加载时使用
- 局部Loading：模块数据刷新时使用
- 无限滚动：列表场景采用滚动加载
- 分页加载：表格、列表类组件采用分页

### 空状态：

内容区域展示为空时，使用空状态图标占位，根据黄金分割比进行放置。空状态应包含：

- 空状态图标（灰色系）
- 提示文字（说明无数据原因）
- 操作入口（如“立即添加”按钮）

### 动效与微交互：

1. 基础动效参数：

| 类型    | 时长        | 缓动曲线                             |
| :---- | :-------- | :------------------------------- |
| 过渡动画  | 200-300ms | ease-in-out                      |
| 悬停反馈  | 150ms     | ease                             |
| 模态框弹出 | 200ms     | cubic-bezier(0.2, 0.9, 0.4, 1.1) |
| 页面切换  | 300ms     | ease                             |

<br />

1. 微交互场景：

- 按钮悬停：轻微颜色变化 + 阴影过渡。
- 输入框聚焦：边框变为品牌色 + 外发光效果。
- 菜单展开/收起：200ms ease过渡。
- 列表项悬停：背景色变化。
